设计一个数据库连接池

2020-03-08 / 3 min read

设计一个数据库连接池

1.1 功能设计

  1. 连接池不与具体的数据库耦合，应具备通用性
  2. 连接池中的连接对象的数量可配置
  3. 提供接口，接入对具体数据库的连接功能

1.2 实现思路

连接池是一个抽象出来的包含多个数据库连接对象的“集合”，这个“集合”不仅保存着一定数量的连接对象，并且还具备对连接对象的管理功能，这些管理功能主要用以保证连接池的可配置性和高可用性。

每个数据库连接对象对应一个线程，多个连接对象则相应的需要多个线程，因此数据库连接池一定是基于多线程技术的。为了高效，需要使用线程池来解决连接对象的创建问题。

为实现连接池不与具体的数据库耦合这一目标，应该设计一个连接对象的接口，将具体的连接对象生成逻辑交由接口的实现类去解决。

连接池要提供一定的可配置项，让用户可根据自身的情况来进行合理的配置，因此可以抽象出一个配置类的对象，用来提供配置功能。

1.3 代码实现

编程语言使用 rust 。

根据上节梳理的实现思路，先定义一个连接池对象，该对象包括了若干成员对象，如下：

struct SharedPool<M>  where  M: ManageConnection {
  config: Config<M::Connection, M::Error>,
  manager: M,
  internals: Mutex<PoolInternals<M::Connection>>,
  cond: Condvar,
}

1.3.1 Config 对象的设计

Config 对象用来提供可配置项。具体代码如下：

pub struct Config<C, E> {
  // 最大连接数量
  pub max_size: u32,
  // 最小闲置连接数量
  pub min_idle: Option<u32>,
  // 
  pub test_on_check_out: bool,
  // 最大生命周期
  pub max_lifetime: Option<Duration>,
  // 闲置超时时长
  pub idle_timeout: Option<Duration>,
  // 建立连接的超时时长 
  pub connection_timeout: Duration,
  // 错误处理器
  pub error_handler: Box<dyn HandleError<E>>,
  // 事件处理器
  pub event_handler: Box<dyn HandleEvent>,
  // 自定义连接项
  pub connection_customizer: Box<dyn CustomizeConnection<C, E>>,
  //  线程池
  pub thread_pool: Arc<ScheduledThreadPool>,
}
